# Onboarding playbook

This note is a compact reminder for the `frontend-onboarding` skill.

## Primary objective

Help a newcomer quickly answer:

1. what this project is,
2. how it starts,
3. how routes are built,
4. where business code lives,
5. where to edit for common frontend changes.

## Preferred answer structure

1. One-sentence conclusion
2. Overall structure
3. Key files
4. Suggested next reading step

## Newcomer-first mindset

Do not optimize for maximum detail first.  
Optimize for:

1. orientation,
2. confidence,
3. correct reading order,
4. safe edit entry points.
